# Object-Oriented Programming

OOP helps model behavior through encapsulation, inheritance, and polymorphism.

%%CW_CARD_START uid:CW-MOCK-001%%

### Polymorphism

Polymorphism lets one interface describe multiple concrete implementations.
A caller can interact with a shared contract while runtime dispatch selects the right behavior.

%%CW_CARD_END uid:CW-MOCK-001%%

## Design Tradeoffs

%%CW_CARD_START uid:CW-MOCK-004%%

### Inheritance vs Composition

Inheritance reuses behavior through an is-a relationship.
Composition reuses behavior by assembling smaller collaborators with a has-a relationship.
Composition usually keeps coupling lower and makes change safer.
